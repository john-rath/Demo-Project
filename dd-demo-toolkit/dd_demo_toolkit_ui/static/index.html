Plain secrets detected in .env

Your .env still holds plain Datadog keys for . Corp policy: secrets live in 1Password, not on disk. Run make migrate-secrets in your terminal for step-by-step migration instructions.

Vertical & overlay

The vertical chooses the demo scenario (devices, services, plugins). Optional overlay layers customer-specific gear on top — e.g. bd on top of healthcare.

Datadog credentials

Enter 1Password references for your keys, not the keys themselves — corp policy. Format: op://<vault>/<item>/<field>. References are written to .env at file mode 0o600; op run resolves them at command time. The save endpoint refuses plain keys.

Simulator settings

Simulator

Runs docker compose up --build --remove-orphans as a child of the UI server. The stack (otel-collector + simulator) inherits the op-resolved env from make ui — secrets stay in 1Password. --build rebuilds the simulator image first (incremental — cached layers are ~2-5s; only edited code triggers real work) so any change to the toolkit code or to a vertical / overlay is picked up automatically on the next Start. Click Stop to send SIGINT (compose's graceful-shutdown signal); a follow-up docker compose down cleans up orphans.
